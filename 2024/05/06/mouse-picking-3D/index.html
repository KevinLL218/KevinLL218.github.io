mouse picking 3D

太长时间没有动脑子，显得思考有些迟钝了。..

3D鼠标拾取功能很久之前研究过，只不过时间太久了，源代码已经遗失了。
今天重新用opengl实现了一遍。并且顺便研究了四元数与欧拉角的关系。(果然还是太难了)

MVP 矩阵

model = translate * rotate * scale;
gl_Position = projection * view * model;

简而言之，MVP矩阵就是上述公式。

model 矩阵

实体p的3维空间坐标如下

$$p=[x,y,z]^{T}$$

该坐标作为其初始坐标，一旦确定就不能改变(理论上)。或者说所有的实体都有一个出生点坐标，然后经过一个变换矩阵将自己投影到空间中.

这里的投影可以认为是将自己的虚影投射到世界中

这里的投影变换矩阵就是model矩阵了

$$model = [translate][rotate][scale]$$

作为矩阵的乘法，自然是从右往左计算了.因此需要先缩放，然后旋转，最后平移。

经过这次变换，p坐标从局部坐标系(Local) 出现在世界坐标系(World)中

view 矩阵

view矩阵和project矩阵都是以摄像机的视点来说的。

在3维空间中放置一个摄像机Camera,那么从摄像机的视角出发观测整个世界的话，需要再次变换。以摄像机为参照物，重建一个摄像机参考系，这个过程就是view变换。

但是把整个世界应用view变换是一个非常困难的计算，但是可以反过来对摄像机应用一个view的反变换，结果也是一样的，只需要变换一次即可。

经过这次变换后，物体从世界坐标系移动到人眼坐标系中

project 矩阵

orthographic / perspective 两种

经过MV映射后的世界，要想显示到屏幕上，还需要进行一次剪裁，就像真实的世界中，整个世界是无限的，但是人眼能看到的也只是一小部分

用人眼来类比就非常明白了：人眼就是摄像机，随着眼睛位置的变化，所观察到的世界也发生变化，这个变化不是说人眼把世界移动到目标位置，而是反过来世界不动，人眼在运动。

人眼能看到的世界也算是一个Perspective视角。

经过这次变换后，物体从人眼坐标系中移动到屏幕坐标系中(screenX, screenY)

3D Picking

picking ray

鼠标拾取的过程，简单来说就是从摄像机的位置射出一道射线，射线击中的第一个物体就是选中的物体。

那么关键的步骤就是计算这个射线。

如果在世界空间中这个计算过程非常简单，这个判断过程也是发生在世界坐标系下的。但是鼠标的坐标是屏幕坐标，计算ray的过程就是屏幕坐标一路逆向到世界坐标。

$$p^T_w=M^{-1}Mp^T$$

对应用过变换矩阵的点再次应用矩阵的逆，就将变换后的点恢复到原来的位置。因为摄像机坐标本身就是世界坐标，所以只需要逆向射线就可以了。

首先创建一个屏幕正中央(0,0)到鼠标指针的屏幕空间的射线:

计算时需要注意，屏幕空间是[0,0,width,height]的坐标系，需要将原点移动到屏幕正中央，得到2维向量，需要升维到4维齐次坐标空间进行计算。

opengl中默认使用的是右手系，z轴指向屏幕外，这里创建的射线要求是指向屏幕内部，因此第三个分量设置为-1，第4个分量就是齐次坐标下的分量，0代表点，1代表向量。

$$ray^T_s = [x, y, -1, 1]^T$$
$$ray^T_e=P^{-1}ray^T_w$$
$$ray^T_w=V^{-1}ray^T_e$$

如此如此这般这般，就得到了从摄像机位置出发，指向鼠标指针在世界空间中的方向上的一条射线。

其实关于这点，很多开发者使用的不是世界坐标而是局部坐标，或本地坐标。还需要对model矩阵求逆，然后再做一次碰撞检测。
